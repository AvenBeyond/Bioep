# 02 项目总计划

## 1. 研究目标
构建一个面向 TCGA-STAD 的多组学亚型分型系统，要求：
- 分型结果具有聚类质量和临床相关性
- 方法可解释、可复现、可答辩
- 包含基线与创新方案的对比及消融

## 2. 阶段划分与产出
1. Phase 1 盘点与目录规范化
   - 产出：目录重构、raw 归档、inventory 文档、路径配置
2. Phase 2 数据概览与项目规划
   - 产出：数据维度与风险文档、总计划文档
3. Phase 3 文献与创新设计
   - 产出：文献综述、创新数学定义、答辩 QA
4. Phase 4 实验蓝图与验证协议
   - 产出：实验流程蓝图、验证协议、答辩讲稿骨架
5. Phase 5 工程骨架初始化
   - 产出：src/tests/main/configs/requirements
6. Phase 6 第一轮自检
   - 产出：smoke test 结果、未决事项登记

## 3. 时间线建议（课程项目节奏）
- 第 1 周：完成 Phase 1-2
- 第 2 周：完成 Phase 3-4
- 第 3 周：完成 baseline 与样本交集分析
- 第 4 周：完成 proposed 方法初版与消融
- 第 5 周：完成临床评估、图表、报告/PPT

## 4. 风险点
- 样本交集过小导致多组学全配对不足
- 超高维特征导致计算负担重
- 生存终点字段完整性不均
- 无外部独立测试集，泛化证据有限

## 5. 依赖关系
- 样本 ID 统一是后续所有分析前置依赖
- 交集统计是是否纳入 RPPA 的决策依赖
- 验证协议是结果解释与答辩措辞依赖
